# HIG: Accessibility (Accesibilidad)

La accesibilidad es un requisito fundamental, no una opción. Diseñamos para que todos los usuarios puedan interactuar con la app.

## Requisitos de Apple
- **Áreas de Toque (Hit Targets):** El tamaño mínimo para elementos interactivos debe ser **44x44 pt**.
- **Texto Dinámico (Dynamic Type):** Soporta escalado de texto para que los usuarios puedan aumentar la fuente en el sistema sin romper el diseño.
- **VoiceOver:** Proporciona etiquetas descriptivas (aria-labels) para todos los elementos de la interfaz.
- **Entradas Alternativas:** Asegura que la funcionalidad principal sea accesible a través de múltiples tipos de interacción.

## Aplicación en Borisao Wiki
- **Aria Labels:** Añadidos a botones de navegación y búsqueda.
- **Soporte de Navegación por Teclado:** Asegurado en el Navbar y Sidebar.
- **Contraste:** Verificado en el fondo abisal con cristal.
